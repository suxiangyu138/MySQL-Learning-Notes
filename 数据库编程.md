# 数据库编程 核心知识点

## 一、数据库编程概述
数据库编程：在数据库内部编写**程序代码**，实现复杂逻辑、数据复用、业务封装，脱离外部代码，直接在MySQL中完成逻辑处理。
核心组件：
1. 变量
2. 流程控制（分支、循环）
3. 存储过程
4. 函数
5. 触发器
6. 游标

> 语法前提：MySQL默认语句结束符是`;`，编程时需临时修改分隔符：
```sql
DELIMITER $$  -- 临时改为 $$
-- 编程代码
$$
DELIMITER ;   -- 改回分号
```

## 二、变量
### 1. 系统变量
由MySQL内置，全局/会话级别
```sql
-- 查看
SHOW VARIABLES;
-- 使用
SELECT @@max_connections;
```

### 2. 自定义变量（用户变量）
会话级别，跨语句生效
```sql
SET @num = 100;
SELECT @num;
```

### 3. 局部变量
仅在存储过程/函数内有效，必须声明
```sql
DECLARE 变量名 数据类型 [DEFAULT 默认值];
```

## 三、流程控制语句
### 1. 分支判断
#### IF 语句
```sql
IF 条件 THEN
  逻辑1;
ELSEIF 条件2 THEN
  逻辑2;
ELSE
  逻辑3;
END IF;
```

#### CASE 语句
```sql
CASE 变量
WHEN 值1 THEN 操作1;
WHEN 值2 THEN 操作2;
ELSE 其他操作;
END CASE;
```

### 2. 循环结构
#### （1）WHILE 循环
```sql
WHILE 条件 DO
  循环体;
END WHILE;
```

#### （2）REPEAT 循环（先执行后判断）
```sql
REPEAT
  循环体;
UNTIL 条件 END REPEAT;
```

#### （3）LOOP 死循环
搭配 `LEAVE` 退出循环

## 四、存储过程【重点】
### 1. 概念
一组**预编译SQL集合**，封装复杂业务逻辑，一次编译、多次调用，效率高。
无返回值，可通过**出入参**传递数据。

### 2. 语法
```sql
DELIMITER $$
CREATE PROCEDURE 过程名(IN 入参,OUT 出参,INOUT 出入参)
BEGIN
  -- 局部变量、流程控制、SQL操作
END $$
DELIMITER ;
```

- `IN`：输入参数（默认）
- `OUT`：输出参数
- `INOUT`：既可输入也可输出

### 3. 调用 & 删除
```sql
-- 调用
CALL 过程名(参数);
-- 删除
DROP PROCEDURE IF EXISTS 过程名;
```

## 五、自定义函数（Function）
### 1. 特点
- **必须有且仅有一个返回值**
- 只能做查询计算，不能增删改数据
- 调用方式：`SELECT 函数名();`

### 2. 语法
```sql
DELIMITER $$
CREATE FUNCTION 函数名(参数 类型) 
RETURNS 返回值类型
BEGIN
  RETURN 结果;
END $$
DELIMITER ;
```

### 3. 函数与存储过程区别（必考）
1. 函数必须返回值；存储过程无返回值
2. 函数适合计算；存储过程适合复杂业务CRUD
3. 函数用`SELECT`调用；存储过程用`CALL`调用

## 六、触发器【高频考点】
### 1. 概念
监听表的 **INSERT / UPDATE / DELETE** 操作，满足条件**自动触发**执行预设SQL。

### 2. 触发时机
- `BEFORE`：操作执行前触发
- `AFTER`：操作执行后触发

### 3. 新旧数据
- `NEW`：新增/修改后的新数据
- `OLD`：修改/删除前的旧数据

### 4. 基础语法
```sql
CREATE TRIGGER 触发器名
触发时机 触发事件
ON 表名 FOR EACH ROW
BEGIN
  触发执行的SQL;
END
```

### 作用
数据同步、日志记录、数据校验、级联更新

---

## 七、游标
作用：逐条读取**查询结果集**，适合多行数据逐行处理。
使用步骤：
1. 声明游标 `DECLARE 游标名 CURSOR FOR 查询语句`
2. 打开游标 `OPEN 游标名`
3. 读取数据 `FETCH 游标名 INTO 变量`
4. 关闭游标 `CLOSE 游标名`

---

## 八、异常处理
通过 `DECLARE HANDLER` 捕获异常，避免程序中断
```sql
DECLARE HANDLER FOR SQLEXCEPTION
BEGIN
  -- 异常处理逻辑
END;
```

## 九、数据库编程优缺点
### 优点
1. SQL封装复用，减少网络传输
2. 预编译执行，性能更高
3. 统一数据库层逻辑，安全可控

### 缺点
1. 数据库压力变大
2. 可读性差、维护困难
3. 不利于集群、分库分表扩展
> 企业开发：**尽量少用存储过程/触发器**，业务逻辑放代码层

---

## 十、极简背诵总结
1. 编程三要素：**变量 + 分支 + 循环**
2. 存储过程：`CALL`调用、支持出入参、处理复杂业务
3. 自定义函数：必须`RETURN`、单返回值、SELECT调用
4. 触发器：监听增删改、自动执行、NEW/OLD
5. 分隔符：`DELIMITER $$` 是数据库编程必备语法
