# 数据库入门 超清晰核心笔记
## 一、什么是数据库
**数据库（DB）**：长期存储在计算机中、**有组织、可共享**的数据集合。
- 作用：永久保存数据、统一管理、高效查询
- 对比：
    - 文件(txt/excel)：数据散乱、查询慢、并发差
    - 数据库：结构化存储、安全、高性能、支持多用户操作

**数据库管理系统（DBMS）**：
用来**创建、管理、操作**数据库的软件，例如：MySQL、Oracle、SQL Server。

**SQL**：结构化查询语言，专门用来操作数据库的统一语法。

---

## 二、数据库三层结构（理解必背）
1. **数据库**：仓库，一个项目对应一个库
2. **数据表**：库中的表格，存放具体数据
3. **数据行/字段**
    - 行（记录）：一条完整数据
    - 字段（列）：数据的属性（姓名、年龄）

层级关系：**数据库 → 数据表 → 字段/行数据**

---

## 三、数据库分类
### 1. 关系型数据库（RDB）
- 特点：以**表、行、列**存储，有约束、结构固定
- 代表：`MySQL`、Oracle、SQL Server、PostgreSQL
- 优势：安全、事务支持、适合常规业务开发

### 2. 非关系型数据库（NoSQL）
- 特点：键值对、文档、列式存储，结构灵活
- 代表：Redis、MongoDB
- 优势：高并发、海量数据、读写快

---

## 四、MySQL 核心概念
1. 开源免费、轻量、企业最常用
2. 默认端口：**3306**
3. 存储引擎：
    - **InnoDB**：默认，支持事务、外键、行级锁（开发首选）
    - MyISAM：不支持事务，查询快，已淘汰

---

## 五、SQL 语言四大分类
1. **DDL 数据定义语言**
   库、表的创建/修改/删除
   `CREATE`、`ALTER`、`DROP`

2. **DML 数据操作语言**
   增删改表中数据
   `INSERT`、`UPDATE`、`DELETE`

3. **DQL 数据查询语言**
   查询数据，开发使用最多
   `SELECT`

4. **DCL 数据控制语言**
   权限、事务管理
   `GRANT`、`COMMIT`、`ROLLBACK`

---

## 六、基础操作命令（入门必会）
### 1. 操作数据库
```sql
-- 查看所有数据库
SHOW DATABASES;

-- 创建数据库
CREATE DATABASE 库名;

-- 使用指定数据库
USE 库名;

-- 删除数据库
DROP DATABASE 库名;
```

### 2. 操作数据表
```sql
-- 查看当前库所有表
SHOW TABLES;

-- 查看表结构
DESC 表名;

-- 删除表
DROP TABLE 表名;
```

### 3. 简单增删改查
```sql
-- 查询
SELECT * FROM 表名;

-- 新增
INSERT INTO 表名(字段1,字段2) VALUES(值1,值2);

-- 修改
UPDATE 表名 SET 字段=值 WHERE 条件;

-- 删除
DELETE FROM 表名 WHERE 条件;
```

## 七、三大完整性（数据库设计核心）
1. **实体完整性**
   主键约束，保证每条数据唯一

2. **域完整性**
   数据类型、非空、默认值，限制字段数据格式

3. **参照完整性**
   外键约束，多表之间数据关联合法

---

## 八、入门必懂常识
1. SQL 语句**不区分大小写**
2. 每条语句结尾加分号 `;`
3. 实际开发严禁：不带条件的 `UPDATE`、`DELETE`
4. 所有数据必须指定合理**数据类型+约束**

---

## 九、学习路线（极简）
1. 数据库、表 基础指令
2. 数据类型 + 六大约束
3. 单表 DQL 查询（条件、排序、分页、聚合、分组）
4. 多表查询、联表
5. 事务、索引、设计规范
