# MySQL 数据库设计 核心知识点（入门+考试+开发通用）

## 一、数据库设计概述
### 1. 概念
数据库设计：根据业务需求，**规划库、表、字段、关系、约束**，合理存储数据，保证：
- 数据完整、安全、冗余少
- 查询高效、易于维护、方便扩展

### 2. 设计流程
1. 需求分析：明确业务、数据、功能
2. 概念设计：E-R模型（实体、属性、关系）
3. 逻辑设计：转为二维表结构、字段、数据类型、约束、表关系
4. 物理设计：建库、建表、索引、引擎选型
5. 维护优化：调优、分表、权限管理

---

## 二、E-R 模型（必考）
### 1. 三大要素
- **实体**：现实事物，对应**数据表**
  例：学生、班级、订单、商品
- **属性**：实体特征，对应**字段**
  例：学生编号、姓名、年龄
- **关系**：实体之间的关联

### 2. 三种表间关系
1. **一对一**
   例：用户 — 用户身份证信息
   设计：一方主键作为另一方外键，或合并为一张表

2. **一对多（最常用）**
   例：班级(1) — 学生(多)、部门 — 员工
   设计：**多方添加外键**，关联一方主键

3. **多对多**
   例：学生 — 课程、商品 — 订单
   设计：**引入中间表**，存放两个实体的外键

## 三、三大范式（数据库设计规范）
作用：**减少数据冗余、避免更新异常、保证数据规范**

### 1. 第一范式 1NF
- 要求：**列不可再分**
- 规则：一个字段只存单一数据，禁止复合字段
- 错误：`地址`字段存「省/市/区」拼接文本
- 正确：拆分为省、市、区 三个独立字段

### 2. 第二范式 2NF
- 前提：满足 1NF
- 要求：**非主键字段完全依赖主键**，消除部分依赖
- 核心：一张表只做一件事

### 3. 第三范式 3NF（开发常用）
- 前提：满足 2NF
- 要求：**消除传递依赖**
- 规则：非主键字段，不能依赖其他非主键字段
- 例：学生表只存班级id，不存班级名称；班级名称放班级表

> 补充：
> 开发中**适当反范式**：为了查询效率，允许少量冗余，减少联表查询。

## 四、表结构设计原则
1. 一库对应一个项目，业务隔离
2. 一张表对应一个实体，职责单一
3. 主键必设：使用 **无意义自增ID / 雪花ID**
4. 字段选型合理
    - 整型：`INT`、`BIGINT`
    - 字符串：短文本`VARCHAR`，长文本`TEXT`
    - 金额：强制 `DECIMAL`
    - 时间：统一 `DATETIME`
5. 合理加约束
    - 必填表：`NOT NULL`
    - 唯一字段：`UNIQUE`
    - 状态字段：`DEFAULT` 默认值
6. 字段统一命名规范
    - 小写+下划线：`user_name`、`create_time`
7. 通用必备字段（企业标准）
```sql
create_time  -- 创建时间
update_time  -- 更新时间
is_delete    -- 逻辑删除(0正常 1删除)
```

## 五、存储引擎选择
### 1. InnoDB（默认、必用）
- 支持事务、行锁、外键、崩溃恢复
- 适合：所有业务表

### 2. MyISAM（淘汰）
- 不支持事务、表锁
- 仅查询快，企业基本不用

---

## 六、主键与外键设计
### 1. 主键设计
- 每张表**必须有主键**
- 推荐：单列自增主键
```sql
id INT PRIMARY KEY AUTO_INCREMENT
```

### 2. 外键设计
- 作用：保证参照完整性
- 现状：**互联网项目基本不用物理外键**，改用代码逻辑控制
- 适用：传统管理系统、教学场景

---

## 七、逻辑删除 vs 物理删除
1. **物理删除**：`DELETE` 真实删除数据，无法恢复
2. **逻辑删除**：加 `is_delete` 字段
    - 0：正常
    - 1：已删除
> 企业开发全部使用**逻辑删除**，保留数据、便于溯源恢复。

---

## 八、索引设计（基础）
设计表时，提前规划索引：
1. 经常 `WHERE`、排序、分页的字段加索引
2. 主键自带主键索引
3. 唯一字段建唯一索引
   作用：大幅提升查询速度

---

## 九、完整设计示例
需求：设计 班级 + 学生 表
1. 班级表（一方）
```sql
CREATE TABLE class(
    id INT PRIMARY KEY AUTO_INCREMENT,
    class_name VARCHAR(30) NOT NULL,
    create_time DATETIME
);
```
2. 学生表（多方，加外键）
```sql
CREATE TABLE student(
    id INT PRIMARY KEY AUTO_INCREMENT,
    stu_name VARCHAR(20) NOT NULL,
    age INT DEFAULT 18,
    class_id INT,
    is_delete TINYINT DEFAULT 0
);
```

## 十、背诵极简总结
1. 设计流程：需求 → E-R模型 → 表结构 → 建库建表
2. 三关系：一对一、一对多、多对多（中间表）
3. 三范式：列不可分、完全依赖、消除传递依赖
4. 建表原则：单表单一职责、字段合理、约束齐全、通用字段、逻辑删除
5. 引擎统一使用 InnoDB
