# MySQL 事务 核心知识点（超精简·考试必背）

## 一、事务概念
**事务**：一组**不可分割**的 SQL 操作，要么**全部执行成功**，要么**全部失败回滚**。
用来保证**数据操作安全、数据一致性**。

---

## 二、事务四大特性 ACID（必考）
1. **A 原子性**
   事务是最小单元，要么全成功，要么全回滚。
2. **C 一致性**
   事务前后，数据整体状态合法、约束完整。
3. **I 隔离性**
   多个事务之间互相隔离，互不干扰。
4. **D 持久性**
   事务提交后，数据永久保存，崩溃不丢失。

---

## 三、事务基本操作语法
```sql
-- 1. 开启事务
START TRANSACTION;

-- 2. 执行多条DML语句(增/删/改)
UPDATE account SET money=money-100 WHERE id=1;
UPDATE account SET money=money+100 WHERE id=2;

-- 3. 正常结束：提交事务
COMMIT;

-- 4. 异常/出错：回滚事务
ROLLBACK;
```

> MySQL InnoDB 引擎**支持事务**；MyISAM 不支持。

---

## 四、事务自动提交
1. MySQL 默认**自动提交事务**
   执行一条 SQL 立刻自动 commit。
2. 关闭自动提交
```sql
SET autocommit = 0;
```

---

## 五、事务并发三大问题
1. **脏读**
   一个事务读取到**另一个事务未提交**的脏数据。
2. **不可重复读**
   同一事务内，**两次读取同一数据**，结果不一致（被其他事务修改并提交）。
3. **幻读**
   同一事务内，范围查询，突然多出/少了别的事务新增删除的数据。

---

## 六、四大隔离级别（由低到高）
| 隔离级别 | 脏读 | 不可重复读 | 幻读 |
|----------|------|------------|------|
| 读未提交 | ✅ 有 | ✅ 有 | ✅ 有 |
| 读已提交 | ❌ 无 | ✅ 有 | ✅ 有 |
| 可重复读（MySQL默认） | ❌ 无 | ❌ 无 | ✅ 有 |
| 串行化 | ❌ 无 | ❌ 无 | ❌ 无 |

### 常用命令
```sql
-- 查看隔离级别
SELECT @@transaction_isolation;

-- 设置隔离级别
SET TRANSACTION ISOLATION LEVEL READ COMMITTED;
```

---

## 七、InnoDB 锁机制（简单理解）
1. **行级锁**：单行加锁，并发高（事务隔离核心）
2. **表级锁**：锁住整张表，并发低
   InnoDB 默认使用**行锁**，只锁定修改行。

---

## 八、核心背诵口诀
1. 事务四大特性：**ACID** 原子、一致、隔离、持久
2. 三大并发问题：**脏读、不可重复读、幻读**
3. MySQL 默认隔离级别：**可重复读**
4. 事务关键命令：`START TRANSACTION`、`COMMIT`、`ROLLBACK`
5. 只有 **InnoDB** 支持事务
