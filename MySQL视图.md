# MySQL 视图（View）核心知识点
## 一、视图概念
**视图：一张虚拟表**，本身**不存储真实数据**，只保存一条 **SELECT 查询语句**。
- 数据来源：来源于真实数据表（基表）
- 本质：封装复杂查询 SQL
- 特点：查视图 = 执行视图内部的查询

---

## 二、视图语法
### 1. 创建视图
```sql
CREATE VIEW 视图名
AS
查询语句;
```

### 2. 查询视图
```sql
-- 像查普通表一样使用
SELECT * FROM 视图名;
```

### 3. 修改视图
```sql
ALTER VIEW 视图名
AS
新查询语句;
```

### 4. 删除视图
```sql
DROP VIEW IF EXISTS 视图名;
```

### 5. 查看视图定义
```sql
SHOW CREATE VIEW 视图名;
```

---

## 三、视图特点
1. 无实际数据，数据实时来自基表
2. 基表数据变化，视图查询结果自动同步
3. 简化复杂多表、联表、分组查询
4. 可以限制字段展示，做**数据安全、权限控制**

---

## 四、视图的增删改
- 简单视图（单表、无聚合、无去重、无分组）：**支持 INSERT / UPDATE / DELETE**
- 复杂视图（含 `GROUP BY`、聚合函数、`DISTINCT`、联表查询）：**不可修改**

> 开发中：视图**只用来查询**，不做增删改。

---

## 五、视图优点
1. **简化SQL**：复杂联表查询封装，重复直接调用
2. **安全隔离**：隐藏敏感字段（手机号、密码），只开放必要字段
3. **逻辑统一**：统一查询口径，修改只需改视图
4. **降低复杂度**：给复杂结果集起别名，方便开发

## 六、视图缺点
1. 不存储数据，每次查询都执行底层SQL，**无性能提升**
2. 基表结构修改，视图可能失效
3. 嵌套视图过多，层级复杂，排错困难

---

## 七、视图 vs 表（必考区分）
1. 表：物理存储数据，占用磁盘空间
2. 视图：虚拟表，只存SQL，不存数据
3. 表可独立增删改；复杂视图只能查

---

## 八、简单示例
```sql
-- 1. 单表创建视图
CREATE VIEW v_student
AS
SELECT id,name,age FROM student WHERE age >= 18;

-- 2. 使用视图
SELECT * FROM v_student;

-- 3. 删除视图
DROP VIEW v_student;
```

---

## 极简背诵版
1. 视图=**虚拟表**，只存查询语句，不存数据
2. 关键字：`CREATE VIEW`
3. 单表简单视图可改，**聚合/分组/多表视图只读**
4. 作用：简化查询、隐藏字段、数据安全
5. 数据随基表实时更新
